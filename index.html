          How the simulation works

          This is the engineer-facing calculation specification for the deployed simulation. It explains how catalog inputs become session predictions and which assumptions must be validated before using the results for a production decision.

          What the prediction means

          Each displayed result is an empirical estimate from 1,000 seeded Monte Carlo sessions per strategy. It is conditional on the entered catalog and the assumptions below; it is not a confidence interval, theoretical guarantee, or calibrated production forecast.

          Calculation pipeline

          1. Expand the catalog table into one exact clip count for each parent title.
          2. Order titles from most clips to fewest as the current proxy for descending RFY rank.
          3. Calculate a parent-title probability for the Personalized or uniform-title strategy.
          4. Divide each title's probability equally among its clips.
          5. Convert clip probabilities into Dirichlet concentration values.
          6. Generate M sessions (1,000 by default) by sampling Gamma scores and selecting the top B unique clips (25 by default).
          7. Compare every session with the union of K randomly selected distinct lookback sessions (three by default).
          8. Aggregate per-session observations into averages, observed extremes, rates, and histograms.

          Personalized Algo (SRA v1)

          For title rank r, normalize the PDF's rank-transformed RFY weight:

          w_r = 1 / sqrt(r + 2) q_i = w_i / sum(w)

          Totally Random Algo

          Every parent title receives equal exposure. This is uniform by title, not by clip.

          q_i = 1 / number_of_titles

          Title weight to clip weight

          If title i has m_i clips, each receives an equal share:

          p_ij = q_i / m_i sum(all p_ij) = 1

          Session generation

          For session size B, draw one Gamma score per clip and take the largest B.

          alpha_ij = B * p_ij g_ij ~ Gamma(alpha_ij, 1) session = TopB(g)

          Metric calculations

          Exact clip duplicates

          |current_clips ∩ union(lookback_clips)|

          Cross-session title repeats

          |current_titles ∩ union(lookback_titles)|

          Within-session title repeats

          session_size - distinct_parent_titles

          Sessions with 2+ clips from a title

          100 * count(within_repeats > 0) / M

          Displayed average

          sum(metric_per_session) / M

          Histogram percentage

          100 * count(metric = bin_value) / M

          Cumulative sessions at threshold t

          count(exact_clip_duplicates <= t)

          Assumptions to validate

          • Clip-rich titles are used as a proxy for the highest RFY ranks.
          • Sessions are independent and use a fixed probability distribution.
          • Lookback sessions are random other sessions, not chronological predecessors.
          • There is no history-aware suppression, freshness, eligibility, or user segmentation.
          • Dirichlet top-K is a heuristic, not exact Plackett-Luce sampling.

          Engineering validation checks

          • Catalog totals equal the expanded title and clip arrays.
          • Title and clip probabilities each sum to one.
          • Concentration values sum to the session size.
          • Every session contains exactly B unique clip indices.
          • Histogram bin counts sum to M.
          • Repeated-title count always equals session size minus distinct titles.
          • Identical inputs and seed reproduce identical outputs.
          • Both strategies receive the same expanded catalog.
          Recommended validation approach

          Reimplement the model independently, test tiny catalogs by inspection, compare multiple seeds and larger run counts, evaluate exact Gumbel top-K as a reference, and compare the simulated distributions with production logs. The full specification is in docs/MODEL.md.
